0:00
Alright, good morning everyone. Thanks for joining. We've got a packed agenda today — we're going to go through the Q2 roadmap, talk through the architecture decisions we landed on last week, and then open it up for questions.
3:00
So the big headline: we're moving to a microservices architecture. I know we've talked about this for a while, but we have executive sign-off, a dedicated migration squad, and a hard deadline of July 31st for the pilot service.
7:00
We evaluated three messaging systems — RabbitMQ, Kafka, and Pulsar. After benchmarking at 10x our projected peak load, Kafka was the clear winner. RabbitMQ hit throughput limits at around 80k messages per second, which we'd blow past in 18 months.
10:00
The payments service is our pilot for microservices migration. It's relatively self-contained, has clear API boundaries, and the team has already written specs for the event schema. If we can migrate payments cleanly, we have a proven playbook for the rest.
14:00
On the API gateway — we're going with Kong. It'll handle auth, rate-limiting, and request routing centrally. Every service that's currently doing its own JWT validation will delegate to the gateway. That's about 30% of our auth code we can delete.
27:00
For on-call, we heard the feedback clearly. 1-week rotations are burning people out. Starting June 1st, we're switching to 2-week rotations with a mandatory recovery week before you go on-call again. We're also adding a shadow rotation for folks who are new to the system.
40:00
Q&A is open. I'll take questions — go ahead Marcus.
41:40
Marcus: What's the fallback plan if the payments migration hits delays? Are we committed to July 31 as a hard date? Sarah: It's a target, not a contract. If we're not confident by July 15th, we push to mid-August. Customer-facing reliability comes first.
